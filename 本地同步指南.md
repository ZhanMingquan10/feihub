# 本地同步指南

## 从 GitHub 同步到本地

### 1. 克隆仓库（首次）
```bash
git clone https://github.com/ZhanMingquan/feihub.git
cd feihub
```

### 2. 更新到最新版本
```bash
git pull origin main
# 或
git pull origin master
```

### 3. 安装依赖
```bash
# 前端依赖
npm install

# 后端依赖
cd backend
npm install
cd ..
```

### 4. 配置环境变量
```bash
# 后端环境变量
cp backend/.env.example backend/.env
# 编辑 backend/.env，填入实际配置
```

### 5. 构建和运行
```bash
# 构建前端
npm run build

# 运行后端（开发模式）
cd backend
npm run dev
```

## 从本地同步到 GitHub

### 1. 检查更改
```bash
git status
```

### 2. 添加更改
```bash
git add .
```

### 3. 提交更改
```bash
git commit -m "描述你的更改"
```

### 4. 推送到 GitHub
```bash
git push origin main
# 或
git push origin master
```

## 注意事项

1. **不要提交敏感信息**
   - `.env` 文件
   - 数据库密码
   - API 密钥

2. **提交前检查**
   - 运行测试
   - 检查构建是否成功
   - 检查是否有语法错误

3. **分支管理**
   - `main` 或 `master`: 生产环境代码
   - `develop`: 开发分支（可选）

## 常见问题

### Q: 推送时提示需要认证
A: 配置 GitHub Personal Access Token 或 SSH 密钥

### Q: 本地有未提交的更改，无法 pull
A: 
```bash
# 暂存更改
git stash

# 拉取更新
git pull

# 恢复更改
git stash pop
```

### Q: 冲突解决
A:
```bash
# 查看冲突文件
git status

# 手动解决冲突后
git add .
git commit -m "解决冲突"
```
