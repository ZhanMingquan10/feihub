# 快速诊断指南

## 问题：Puppeteer 没有工作

你看到的内容是简化方案的提示，说明 Puppeteer 没有正常工作。

## 🔍 诊断步骤

### 第一步：查看后端启动日志

**重新启动后端服务**，查看启动时的日志：

1. 停止后端服务（`Ctrl+C`）
2. 重新运行 `backend/完整启动.bat`
3. **查看启动日志**，应该看到：

**如果 Puppeteer 可用：**
```
[启动] 正在检查 Puppeteer 状态...
[启动] ✅ puppeteer-core 已安装
[启动] ✅ Puppeteer 可用，将使用 Puppeteer 方案
```

**如果 Puppeteer 不可用：**
```
[启动] 正在检查 Puppeteer 状态...
[启动] ⚠️ puppeteer-core 未安装
[启动] 请运行: npm install puppeteer-core
```

或

```
[启动] ⚠️ Puppeteer 加载失败: ...
[启动] 将使用 cheerio 方案（可能无法获取动态内容）
```

### 第二步：检查依赖

运行 `backend/简单检查.bat`（已修复，不会闪退）

### 第三步：如果 puppeteer-core 未安装

```bash
cd backend
npm install puppeteer-core
```

### 第四步：检查 Chrome

确保已安装 Google Chrome 浏览器：
- 下载：https://www.google.com/chrome/
- 安装后重启后端服务

## 📋 请告诉我

重启后端服务后，把启动日志发给我，特别是：
1. `[启动]` 开头的日志
2. 是否有 `✅ Puppeteer 可用` 或 `⚠️` 的提示
3. 提交文档时的 `[爬取]` 和 `[Puppeteer]` 日志

这样我可以帮你定位问题！
