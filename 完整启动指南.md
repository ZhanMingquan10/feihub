前端会在 http://localhost:5173 启动

   - 访问：http://localhost:5173

3. **测试提交文档**

## 🐛 如果遇到问题

### 前端无法连接后端
- 检查后端服务是否运行（http://localhost:4000/health）
- 检查 `src/lib/api.ts` 中的 API 地址是否正确

### 提交文档失败
- 检查后端控制台是否有错误信息
- 检查 DeepSeek API Key 是否正确配置
- 检查 Docker 容器是否正常运行

### 数据库连接失败
- 检查 Docker Desktop 是否运行
- 运行：`docker-compose up -d`（在 backend 目录）

## 📝 重要提示

1. **保持后端服务运行**
   - 后端服务窗口不要关闭
   - 如果关闭了，重新运行 `backend/完整启动.bat`

2. **保持 Docker 运行**
   - Docker Desktop 需要一直运行
   - 如果关闭了，数据库和 Redis 会停止

3. **API Key 安全**
   - DeepSeek API Key 已配置在 `backend/.env` 中
   - 不要将 `.env` 文件提交到 Git
