# 修复客服图片显示 - 部署步骤

## 🔍 问题分析

`kefu.png` 图片在项目根目录，但代码中使用 `/kefu.png`。在 Vite 中，静态资源需要放在 `public` 目录，构建后才会复制到 `dist` 目录。

## 🚀 解决方案

### 方法一：在服务器上直接修复（推荐）

在服务器上执行：

```bash
cd /www/wwwroot/feihub

# 创建 public 目录（如果不存在）
mkdir -p public

# 复制图片到 public 目录
cp kefu.png public/kefu.png

# 验证
ls -la public/kefu.png

# 重新构建前端
npm run build

# 验证构建后的文件
ls -la dist/kefu.png

# 重载 Nginx
nginx -s reload
```

### 方法二：在本地修复后提交

**本地操作：**
1. 创建 `public` 目录
2. 将 `kefu.png` 复制到 `public/kefu.png`
3. 提交到 GitHub

**服务器操作：**
```bash
cd /www/wwwroot/feihub
git pull origin main
npm run build
nginx -s reload
```

## ✅ 验证

1. 访问网站
2. 点击"客服"按钮
3. 查看弹窗中的图片是否正常显示

## 📝 如果图片还是不显示

检查以下几点：

1. **检查构建后的文件是否存在**：
   ```bash
   ls -la /www/wwwroot/feihub/dist/kefu.png
   ```

2. **检查 Nginx 配置**（确保静态文件可以访问）：
   ```bash
   grep -A 5 "location" /www/server/panel/vhost/nginx/feihub.top.conf
   ```

3. **检查浏览器控制台**（F12）：
   - 查看 Network 标签，看图片请求是否 404
   - 查看 Console 标签，看是否有错误

4. **直接访问图片 URL**：
   - 在浏览器中访问：`http://feihub.top/kefu.png`
   - 或：`http://121.40.214.130/kefu.png`
